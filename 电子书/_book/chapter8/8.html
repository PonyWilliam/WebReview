知识点

event loop(事件循环/事件轮询)

基础知识

  1. js是单线程运行的
  2. 异步要基于回调来实现
  3. event loop是异步回调的实现原理

    js如何执行？

  4. 从前到后，一行一行执行
  5. 如果某一行执行报错，则停止下面代码的执行
  6. 先把同步代码执行完，再执行异步

    总结：

  7. 同步代码，一行一行放在call back执行
  8. 遇到异步，会先“记录”下，等待时机（定时、网络请求）
  9. 时机一到，移动到callback Queue
  10. 如果call stack为空，event loop开始工作
  11. 轮训查找callback queue，如有则移动到call stack执行
  12. 如何继续轮寻查找（永动机）

    dom事件和event loop

    知识点

  13. 异步使用回调，基于event loop
  14. DOM事件也使用回调，基于event loop

    Promise

  15. 三种状态
    1. pending 正在进行
    2. resolved 已被解决(新标准采用fufilled)
    3. rejected 已被拒绝
    4. pending -> resloved
    5. pending -> rejected
    6. 这些状态不可逆
  16. 状态的表现和变化
    1. pending状态不会触发then 和 catch
    2. resolved状态会触发后续的then回调函数
    3. rejected状态会触发后续的catch回调函数
  17. then和catch对状态的影响
    1. then正常返回resolved，里面有报错则返回rejected
    2. catch正常返回resolved，里面有报错则返回rejected

      tips:

  18. 触发顺序由嵌套层数确定，嵌套层数越多，触发越靠后。

    async/await

    知识点

  19. 异步回调 callback hell
  20. Promise then catch 链式调用，但也基于回调函数
  21. async/await式同步语法，彻底消灭回调函数。

async/await 和 promise的关系

概述

  1. async/await是消灭异步回调的终极武器
  2. 和promise并不冲突
  3. 两者相辅相成

    总结

  4. 执行async返回是promise对象
  5. await相当于promise的then
  6. try...catch可捕获异常，代替了promise的catch

异步本质

概述

  1. async/await 是消灭异步回调的终极武器
  2. js还是单线程，还得有异步，还得基于event loop
  3. async/await 知识一个语法糖

    for ... of

  4. for ... in 以及forEach for 是常规的同步遍历
  5. for ... of常用于异步遍历

宏任务与微任务

知识点

  1. 什么是宏任务，什么是微任务
  2. event loop 和dom渲染
  3. 微任务和宏任务的区别

    知识点1：

  4. 宏任务：setTImeout，setInterval，ajax，dom事件
  5. 微任务：Promise async/await
  6. 微任务执行时机比微任务早（先记住）
  7. 区别：宏任务：DOM渲染后触发、微任务：DOM渲染前先触发

    知识点2：

  8. 再看一次event loop过程
  9. js和dom渲染共享一个线程
  10. 当call stack空闲，会尝试dom渲染，然后将dom渲染，然后触发event loop

    知识点3

    微任务是ES6语法规定的，会单独有一个
